# 市場總覽 · 自動更新版（GitHub Pages + Actions）部署說明

這一版會**每天早上 08:00 由 GitHub 的伺服器自己更新**，完全不靠 Cowork、不靠我的環境。
數字（指數、三大法人、融資融券、行情）自動抓官方/免費來源；新聞與川普發言由 Google News＋你的 Gemini 金鑰自動整理。

---

## 檔案清單（整包上傳到同一個 repo 根目錄）
```
build.py                 主程式：抓資料＋叫 Gemini＋產生 index.html
template.html            網頁樣板（含 %%TOKEN%% 佔位，build.py 會填）
requirements.txt         Python 套件清單
manifest.webmanifest     PWA 設定
sw.js                    離線快取
icon-192.png / 512.png   App 圖示
.github/workflows/update.yml   每日排程設定
（index.html 與 data.json 會由機器人自動產生，不用你自己放）
```

## 設定步驟（一次性，約 20–30 分鐘，我會陪你）

### 1. 建 GitHub 帳號與 repo
1. 到 github.com 註冊免費帳號並登入。
2. 右上「＋」→ New repository。名稱例如 `market-brief`，選 **Public**，按 Create。

### 2. 上傳檔案
1. 進到剛建的 repo，點 **Add file → Upload files**。
2. 把上面清單的檔案整包拖進去（含 `.github/workflows/update.yml` 這個資料夾結構）。
3. 按 **Commit changes**。

### 3. 放入 Gemini 金鑰（Secret，安全）
1. repo → **Settings → Secrets and variables → Actions → New repository secret**。
2. Name 填 **`GEMINI_API_KEY`**，Secret 貼上你的 Gemini 金鑰，按 Add secret。
   （金鑰只放在這個加密欄位，不會出現在網頁或程式碼裡。）

### 4. 開啟 GitHub Pages
1. repo → **Settings → Pages**。
2. Source 選 **Deploy from a branch**，Branch 選 **main**、資料夾 **/(root)**，Save。
3. 稍等 1–2 分鐘，會出現你的網址：`https://<你的帳號>.github.io/market-brief/`

### 5. 第一次手動觸發（產生 index.html）
1. repo → **Actions** 分頁 → 左側 **daily-build** → 右側 **Run workflow** → Run。
2. 跑完（約 1–2 分鐘）後，repo 會多出 `index.html` 與 `data.json`。
3. 打開上面的 Pages 網址即可看到；手機開它 →「加到主畫面」就是 App。

之後每天早上 08:00 會自動重跑更新。想立即更新，就到 Actions 按一次 **Run workflow**。

## 手動更新怎麼運作
- 網頁上的「↻ 更新」按鈕＝**重新載入最新已發佈版本**（不會即時重抓，因為即時重抓要動用金鑰，放進網頁不安全）。
- 要「馬上重抓最新資料」＝到 **Actions → Run workflow** 按一下（約 1–2 分鐘），再重整網頁。

## 五日歷史怎麼來
三大法人、融資融券的五日表，是靠 `data.json` **每天累積**——剛上線時只有 1–2 天，之後每天自動補一格，約一週後就會填滿五日。這是刻意設計，不用估算假數字。

## 老實的注意事項
- 這是**第一版**。抓資料的小程式依賴對方網站格式，第一次跑可能有某個來源（最常見是期交所表格或某個 Stooq 代碼）解析不到 → 那個區塊會顯示「待更新／待補」，頁面不會壞。
- 若某區塊一直抓不到，把該次 **Actions 執行記錄（log）** 貼給我，我幫你修對應的 `fetch_*` 函式或 Stooq 代碼。
- Gemini 免費額度每天足夠這個用途；金鑰若換要記得更新 Secret。
